= claude-memory-files

Curated, modular memory and instruction files for https://claude.com/claude-code[Claude Code], maintained by Ribose for developers across our orgs (`riboseinc/*`, `metanorma/*`, `lutaml/*`, `plurimath/*`, `relaton/*`).

== Audience signposts

* **Consumer** — install a file someone wrote → link:getting-started/installation/[Installation] · link:guides/cherry-pick-via-curl/[Cherry-pick via curl] · link:guides/using-slash-commands/[Slash commands].
* **Contributor** — share a file you've written → link:concepts/scope-rubric/[Pick your scope] · link:guides/submitting-a-file/[Submit a file] · link:reference/frontmatter-schema/[Frontmatter contract].
* **Maintainer** — review PRs or ship releases → link:reference/validator-rules/[Validator rules] · link:concepts/mechanism-and-data/[Mechanism/data pairings] · link:roadmap/v1.1-and-v2/[Roadmap].

== Quick start

Install one file via curl (zero setup required):

[source,bash]
----
curl -fsSL https://raw.githubusercontent.com/riboseinc/claude-memory-files/main/tools/install.sh \
  | bash -s -- instructions/github-pr-title-issue-link.md
----

For the durable workflow (manage installs, list outdated, submit your own), install the companion plugin once:

[source]
----
/plugin marketplace add riboseinc/claude-memory-files
/plugin install memory-files-tools@claude-memory-files
----

Then `/list-memory-files` to browse, `/install-memory-file <slug>` to install. See link:guides/using-slash-commands/[Using the slash commands].

== Five-level learning path

. **Install one file** — link:getting-started/installation/[curl one-liner], verify it loads in your next Claude Code session.
. **Install several** — mix `universal` and `team` scope files; see link:concepts/scope-rubric/[scope rubric].
. **Customise a `personal-share`** — link:guides/customise-personal-share/[walkthrough] using `user-maintained-gems`.
. **Submit your own** — link:guides/submitting-a-file/[guide] for `/submit-memory-file`.
. **Maintain a category** — tag-team review with the maintainer; help curate seed batches.

== What this repo deliberately does NOT host

Slash commands, subagents, skills, MCP server configs, output styles, and keybindings are abundantly catalogued by https://github.com/hesreallyhim/awesome-claude-code[awesome-claude-code] and similar aggregators. We focus on the white-space categories: memory, instructions, settings fragments, project CLAUDE.md templates, and path-scoped rules. See the https://github.com/riboseinc/claude-memory-files#further-reading-and-exemplar-listings[README "Further reading"] for the curated aggregator list.
